Claude writes code. Looper makes it work.

The only agentic quality loop for Claude Code. Install once, get code that compiles, passes lint, and ships with green tests. Every time.

You prompt Claude. Claude writes code. You run the tests. They fail. You paste the errors back. Claude fixes one thing, breaks another. You paste again. Three rounds later you're doing Claude's QA by hand.

Looper intercepts Claude's stop signal and runs your quality gates automatically. If anything fails, Claude gets the errors and keeps going. No copy-paste. No babysitting. The loop runs until your code is clean or the budget runs out.

One command. Zero config.
claude plugin install looper@claude-plugins-official

Requires jq. Local dev: claude --plugin-dir /path/to/looper

How it works

1
Claude edits your code. Looper tracks every file touched and runs fast per-file checks (formatting, linting) after each edit.
2
Claude says "done." Looper runs your quality gates: typecheck, lint, test, coverage. Each gate is a shell command that exits 0 on pass.
3
Gates fail? Claude gets the failure output and automatically takes another pass. Gates pass? The loop ends. Clean code delivered.

Detects your stack. Configures itself.

On first session, Looper checks for marker files and picks the right preset. No JSON to write, no commands to look up.

Build your own packages

Quality gates are just the default package. The kernel dispatches to any package you write. TDD cycles, security audits, documentation checks -- three files and a JSON manifest.

packages/my-package/
  package.json            # name, phase, tool matchers
  hooks/
    session-start.sh      # inject context at session start
    stop.sh               # evaluate, exit 0 (done) or 2 (continue)

Multiple packages compose. Core-phase packages run first. Post-phase packages only run after core is satisfied. The kernel aggregates votes -- any package can keep the loop going.

Under the hood

A ~500-line bash kernel dispatches four Claude Code hooks to pluggable packages. No npm dependencies. No build step. Just shell scripts and jq.

SessionStart  init state, inject gate list into Claude's context
PreToolUse    track files, inject pass counter, enforce budget
PostToolUse   run per-file checks (format, lint) after each edit
Stop          evaluate gates, score, vote continue/done

Circuit breakers prevent infinite loops: a re-entry guard, an iteration budget (default 10), and a two-phase stop model that skips secondary checks when primary gates fail.

Configure what matters

Everything lives in .claude/looper.json. Per-project. Version-controlled. Human-readable.

{
  "max_iterations": 10,
  "packages": ["quality-gates"],
  "quality-gates": {
    "gates": [
      { "name": "typecheck", "command": "npx tsc --noEmit", "weight": 30 },
      { "name": "lint",      "command": "npx eslint .",     "weight": 20 },
      { "name": "test",      "command": "npm test",         "weight": 30 }
    ],
    "context": [
      "This project uses React 19 with Server Components.",
      "Never modify the public API contract in docs/api.md."
    ]
  }
}

Gates support skip_if_missing, run_when file patterns, per-gate timeouts, optional vs required flags, and custom coaching messages.
